# dotfiles

Cross-platform dotfiles repo for macOS and Linux (Raspberry Pi).

## Repo structure

```
dotfiles/
├── shared/shell/         # Cross-platform zsh configs
│   ├── .zshenv           → ~/.zshenv (ZDOTDIR + XDG)
│   ├── .zshrc            → ~/.config/zsh/.zshrc
│   └── aliases.zsh       → ~/.config/zsh/aliases.zsh
├── shared/git/
│   ├── .gitconfig        → ~/.gitconfig (uses [include] for platform overrides)
│   └── ignore            → ~/.config/git/ignore
├── macos/shell/.zprofile → ~/.zprofile (Homebrew + nvm)
├── macos/hammerspoon/    → ~/.hammerspoon/ (window management)
├── macos/launchd/        → ~/Library/LaunchAgents/ (COPIES, not symlinks)
├── macos/scripts/        → ~/.local/bin/* (automation scripts)
├── macos/defaults.sh     # macOS system preferences (opt-in)
├── linux/shell/          # .zprofile, .p10k.zsh
├── linux/scripts/        → /usr/local/bin/* (Pi server scripts)
├── linux/systemd/        → /etc/systemd/system/* (COPIES, not symlinks)
├── linux/nginx/          → /etc/nginx/sites-available/* (reverse proxy)
├── linux/config/         → various /etc/ paths (server configs)
├── linux/install.sh      # Pi server deployment (needs sudo)
├── bin/                  → ~/.local/bin/* (cross-platform utilities)
├── bootstrap.sh          # OS-detecting installer (user-level)
├── Brewfile              # Homebrew packages (macOS)
└── dotfiles.env.example  # Template for private config
```

## Conventions

- Symlinks for configs and scripts, copies for LaunchAgent plists and systemd units
- User configs: `bootstrap.sh` (no sudo) — shell, git, bin scripts
- Server infra: `linux/install.sh` (needs sudo) — scripts, systemd, nginx, configs
- Scripts deploy to `~/.local/bin/` (user) or `/usr/local/bin/` (server)
- Platform branching: `if [[ "$OSTYPE" == darwin* ]]`
- Private values (NTFY_URL, PI_HOST) sourced from `~/.config/dotfiles/env`
- Git platform overrides via `[include] path = ~/.config/git/local.gitconfig`
- 2-space indentation, `set -euo pipefail` for bash scripts
- Zsh scripts use `#!/bin/zsh`, bash scripts use `#!/bin/bash`

## Deployment

### User configs (both platforms)
Run `./bootstrap.sh` — detects macOS vs Linux, deploys shell/git/bin configs.
Idempotent: backs up existing files, overwrites symlinks.

### Pi server infrastructure
Run `sudo ./linux/install.sh` — deploys scripts, systemd units, nginx configs,
and server configs. Separate from bootstrap.sh because it needs root.

### Dashboard
Separate repo: `polybjorn/pi-dashboard` at `~/repositories/pi-dashboard/`.
Symlinked to `/var/www/pi-dashboard/` by `linux/install.sh`.

## Key paths

- Repo: `~/repositories/dotfiles/`
- Dashboard: `~/repositories/pi-dashboard/`
- Private config: `~/.config/dotfiles/env` (gitignored via dotfiles.env)
- Platform git overrides: `~/.config/git/local.gitconfig` (generated by bootstrap)
- User scripts: `~/.local/bin/` (symlinks to repo)
- Server scripts: `/usr/local/bin/` (symlinks to repo, Pi only)
- ZDOTDIR: `~/.config/zsh/`
